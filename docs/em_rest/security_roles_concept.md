In this article

You can provide Enterprise Manager accounts with permissions by assigning them security roles. In the Veeam Backup Enterprise Manager web UI, you can assign one of the following roles to the account: Portal Administrator, Portal User or Restore Operator. For details on security roles in the Enterprise Manager web UI, see the [Configuring Accounts and Roles](https://helpcenter.veeam.com/docs/backup/em/veeam_backup_em_roles.html?ver=120) section of the Veeam Backup Enterprise Manager User Guide.

Veeam Backup Enterprise Manager REST API implements the same idea but in a slightly different way. The REST API offers the following set of security roles:

* [Portal Administrator](#PortalAdministrator)
* [Portal User](#PortalUser)
* [VM Restore Operator](#VMRestoreOperator)
* [File Restore Operator](#FileRestoreOperator)
* [Exchange Restore Operator](#ExchangeRestoreOperator)
* [Oracle Restore Operator](#ExchangeRestoreOperator)
* [SQL Restore Operator](#ExchangeRestoreOperator)

You can assign multiple roles to an account. The set of roles depends on the resources and operations that you want to be available for the account. For details on how to assign a security role, see [POST /security/accounts/{ID}/roles](post_security_accounts_id_roles.md).
