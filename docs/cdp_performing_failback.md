# Performing Failback


For more information on failback, see [Failover and Failback for CDP](cdp_failover_failback.md) and [Failback](cdp_failback.md).

To perform failback, use the Failback wizard.

1. [Check prerequisites](cdp_failback_byb.md).
2. [Launch the Failback wizard](cdp_failback_launch.md).
3. [Select replicas](cdp_failback_replicas.md).
4. [Select a target destination](cdp_failback_destination.md).
5. [Select a host](cdp_failback_host.md).
6. [Select a resource pool](cdp_failback_resource_pool.md).
7. [Select a datastore](cdp_failback_datastore.md).
8. [Select a folder](cdp_failback_folder.md).
9. [Configure network mapping](cdp_failover_network.md).
10. [Map replicas to recovered VMs](cdp_failback_map.md).
11. [Schedule switching to production](cdp_failback_schedule.md).
12. [Finish working with the wizard](cdp_failback_summary.md).
