# Public Cloud

In this article

You can perform protection and recovery tasks for the following public cloud platforms:

* [Amazon Web Services](amazon_web_services.md). You can create backups of EC2 instances, RDS resources, DynamoDB tables, EFS and FSx file systems, Redshift Serverless namespaces and VPC configurations. Recover EC2 instances to the original platform and other platforms, recover folders and volumes, recover RDS DB instances, PostgreSQL DB instances, Aurora DB clusters and also perform other operations.
* [Microsoft Azure](microsoft_azure.md). You can create backups of Azure VMs, Azure SQL databases, Azure Cosmos DB accounts, Azure file shares and Azure virtual network configurations. Recover Azure VMs to the original platform and other platforms, recover individual virtual disks, and guest OS files and folders, Azure SQL databases and also perform other operations.
* [Google Cloud](google_cloud.md). You can create backups of VM instances, Cloud SQL instances and Cloud Spanner instances. Recover VM instances to the original platform and other platforms, recover individual persistent disks, and guest OS files and folders, and also perform other operations. Note that this functionality is available on Microsoft Windows=based backup server only.
* [Kubernetes](kubernetes.md). You can manage Kasten instances and policies. Recover Kasten backups to the original platform, export disks from backups and perform First Class Disk Recovery.
