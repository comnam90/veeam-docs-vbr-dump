# Restoring Point-in-Time State

In this article

To restore multiple MongoDB collections to a point-in-time state, use the Restore wizard.

1. [Launch the Restore wizard](vemdb_rs_restore_collection_multiple_pit_wizard.md).
2. [Specify restore point](vemdb_rs_restore_collection_multiple_pit_specify_restore_point.md).
3. [Specify restore settings](vemdb_rs_restore_collection_multiple_pit_specify_restore_settings.md).
4. [Specify the replica set credentials](vemdb_rs_restore_collection_multiple_pit_specify_rs_credentials.md).
5. [Specify primary node credentials](vemdb_rs_restore_collection_multiple_pit_specify_primary_node_credentials.md).
6. [Select databases](vemdb_rs_restore_collection_multiple_pit_select_databases.md).
7. [Specify collections](vemdb_rs_restore_collection_multiple_pit_select_collections.md).

|  |
| --- |
| Note |
| Point-in-time restore is available only if oplog backups exist. For more information, see [Required Job Settings](vemdb_job_settings.md). |
