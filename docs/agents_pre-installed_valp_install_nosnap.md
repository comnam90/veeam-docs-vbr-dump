# Installing Nosnap Veeam Agent for Linux on Power from Generated Setup Files


To install nosnap Veeam Agent for Linux on Power on a computer added to a protection group for pre-installed Veeam Agents, do the following:

1. Upload the setup files generated by Veeam Backup & Replication to the computer you want to protect — for example, to a directory on a local drive or USB drive, or to a network shared folder.
2. On the computer you want to protect, navigate to the directory where you have saved the packages and install Veeam Agent:

* [Installing nosnap Veeam Agent for Linux on Power in RHEL](#rhel)
* [Installing nosnap Veeam Agent for Linux on Power in SLES](#sles)

Installing Nosnap Veeam Agent for Linux on Power in RHEL

To install nosnap Veeam Agent for Linux on Power, use the following commands:

For RHEL 8.4, 8.6, 8.8 and 8.10

|  |
| --- |
| rpm -i <...>/veeam-libs-13.0.1.404-1.ppc64le.rpm |

For RHEL 9.0, 9.2, 9.4 and 9.6

|  |
| --- |
| rpm -i <...>/veeam-libs-13.0.1.404-1.ppc64le.rpm |

For RHEL 10

|  |
| --- |
| rpm -i <...>/veeam-libs-13.0.1.404-1.ppc64le.rpm rpm -i <...>/veeam-nosnap-13.0.1.404-1.el10.ppc64le.rpm |

where <...> is the path to the directory where you have saved the Veeam Agent packages.

Installing Nosnap Veeam Agent for Linux on Power in SLES

To install nosnap Veeam Agent for Linux on Power, use the following commands:

For SLES for SAP 12 SP5

|  |
| --- |
| zypper in <...>/veeam-libs-13.0.1.404-1.ppc64le.rpm zypper in <...>/veeam-nosnap-13.0.1.404-1.sle12.ppc64le.rpm |

For SLES / SLES for SAP 15 SP3 – SP7 and 16.0

|  |
| --- |
| zypper in <...>/veeam-libs-13.0.1.404-1.ppc64le.rpm zypper in <...>/veeam-nosnap-13.0.1.404-1.sle15.ppc64le.rpm |

where <...> is the path to the directory where you have saved the Veeam Agent packages.
