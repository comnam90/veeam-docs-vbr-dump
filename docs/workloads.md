# Workloads


Veeam Backup & Replication is a data protection solution designed to support a [wide range of platforms](platform_support.md), including virtual, physical, cloud platforms, and others. Each platform has unique data protection operations and workflows.

This section describes features and workflows specific to each platform:

* [VMware vSphere](vmware_vsphere.md)
* [VMware Cloud Director](vcloud_director.md)
* [Microsoft Hyper-V](ms_hyperv.md)
* [Nutanix AHV](nutanix_ahv.md)
* [Oracle Linux Virtualization Manager and Red Hat Virtualization](olvm_rhv.md)\*
* [Proxmox Virtual Environment](proxmox_ve.md)
* [Microsoft Entra ID](entra_id.md)
* [Scale Computing HyperCore](sc_hypercore.md)
* [Unstructured Data](unstructured_data_backup.md)
* [Databases and Enterprise Applications](protect_applications.md)
* [Veeam Agent Backup](protect_comp.md)
* [Amazon Web Services](amazon_web_services.md)
* [Microsoft Azure](microsoft_azure.md)
* [Google Cloud](google_cloud.md)\*
* [Kubernetes](kubernetes.md)

In addition to platform-specific features, Veeam Backup & Replication also provides features that can be used with all platforms. For more information, see [Platform-Independent Features](general_vbr_features.md).

\*This feature is available for Microsoft Windows-based backup servers.
