# Managing Tenant Accounts


The SP can perform the following actions with tenant accounts:

* [Disable and enable tenant accounts](cloud_connect_disable_account.md).
* [Rename tenant accounts](renaming_user_accounts.md).
* [Change passwords for tenant accounts](cc_change_password_sp.md).
* [Change resource allocation for tenant accounts](cloud_connect_edit_tenant_account.md).
* [Redeploy network extension appliances for tenant accounts](redeploy_appliance.md).
* [View resource consumption by tenant machines](cloud_connect_view.md).
* [Delete tenant accounts](cloud_connect_delete_account.md).

|  |
| --- |
| Note |
| By default, in case the SP backup server is managed by Veeam Service Provider Console version 5.0 or later, you cannot manage tenant accounts in Veeam Backup & Replication. You can change this setting in Veeam Service Provider Console. To learn more, see the [Managing Veeam Cloud Connect Servers](https://helpcenter.veeam.com/docs/vac/provider_admin/connect_cloud_server.html?ver=9) section in the Guide for Service Providers. |
