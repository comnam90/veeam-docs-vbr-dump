# Creating Universal CDP Policies

In this article

To protect workloads with universal CDP, you must configure a CDP policy. The CDP policy defines which workloads to protect, where to store replicas, how often create short-term and long-term restore points, and so on. One CDP policy can process one or multiple workloads.

To create a CDP policy use the New Universal CDP Policy wizard.

1. [Check prerequisites](uni_cdp_policy_byb.md).
2. [Launch the wizard](uni_cdp_policy_launch.md).
3. [Specify a policy name and advanced settings](uni_cdp_policy_name.md).
4. [Select workloads to replicate and their processing order](uni_cdp_policy_workloads.md).
5. [Exclude objects from processing](uni_cdp_policy_exclude.md).
6. [Select the replica destination](uni_cdp_policy_destination.md).
7. [Configure network mapping](uni_cdp_policy_network.md).
8. [Specify data transfer and replica settings](uni_cdp_policy_data_transfer.md).
9. [Specify notification settings](uni_cdp_policy_notification.md).
10. [Specify the replication schedule](uni_cdp_policy_schedule.md).
11. [Finish working with the wizard](uni_cdp_policy_summary.md).
