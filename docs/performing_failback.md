# Performing Failback


For more information on failback, see [Failback](failback.md) and [Failover and Failback for Replication](failover_failback.md).

To switch back to the source VM, use the Failback wizard.

1. [Check prerequisites](failback_before_you_begin.md).
2. [Launch the Failback wizard](failback_launch_vm.md).
3. [Select VM replicas to fail back](failback_replica_vm.md).
4. [Select a failback destination](failback_destination_vm.md).
5. [Select a failback target host](failback_host_vm.md).
6. [Select a failback target resource pool](failback_pool_vm.md).
7. [Select a failback target datastore](failback_datastore_vm.md).
8. [Select a failback target folder](failback_folder_vm.md).
9. [Select a target network](failback_network_vm.md).
10. [Map the replica to the restored VM](failback_target_vm.md).
11. [Review summary and finish working with the wizard](failback_summary_vm.md).
