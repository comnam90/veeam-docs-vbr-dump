# Performing Failback


For more information on failback, see [Failover and Failback for Universal CDP](uni_cdp_failover_failback.md) and [Failback](uni_cdp_failback.md).

To perform failback, use the CDP Replica Failback wizard.

1. [Check prerequisites](uni_cdp_failback_byb.md).
2. [Launch the wizard](uni_cdp_failback_launch.md).
3. [Select replicas](uni_cdp_failback_replicas.md).
4. [Select a host](uni_cdp_failback_host.md).
5. [Select a resource pool](uni_cdp_failback_resource_pool.md).
6. [Select a datastore](uni_cdp_failback_datastore.md).
7. [Select a folder](uni_cdp_failback_folder.md).
8. [Configure network mapping](uni_cdp_failover_network.md).
9. [Schedule switching to production](uni_cdp_failback_schedule.md).
10. [Finish working with the wizard](uni_cdp_failback_summary.md).
