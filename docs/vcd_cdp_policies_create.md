# Creating Cloud Director CDP Policies


To protect vApps with CDP, you must configure a VMware Cloud Director CDP policy. The CDP policy defines which vApps to protect, where to store replicas, how often create short-term and long-term restore points, and so on. One CDP policy can process one or multiple vApps.

To create a CDP policy, use the New VMware Cloud Director CDP Policy wizard.

1. [Launch the New VMware Cloud Director CDP Policy wizard](vcd_cdp_policy_launch.md).
2. [Specify a policy name and advanced settings](vcd_cdp_policy_name.md).
3. [Select vApps to replicate](vcd_cdp_policy_vapps.md).
4. [Exclude objects](vcd_cdp_policy_exclude.md).
5. [Specify vApp processing order](vcd_cdp_policy_order.md).
6. [Select destination for replicas](vcd_cdp_policy_destination.md).
7. [Configure network mapping](vcd_cdp_policy_network.md).
8. [Configure seeding and mapping](vcd_cdp_policy_seeding_mapping.md).
9. [Specify data transfer and replica settings](vcd_cdp_policy_settings.md).
10. [Specify notification settings](vcd_cdp_policy_notification.md).
11. [Configure a replication schedule](vcd_cdp_policy_schedule.md).
12. [Specify guest processing settings](vcd_cdp_policy_guest.md).
13. [Finish working with the wizard](vcd_cdp_policy_summary.md).
